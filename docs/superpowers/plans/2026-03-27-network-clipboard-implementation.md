# 网络剪贴板实现计划

## 概述
基于设计文档 `2026-03-27-network-clipboard-design.md` 实现网络剪贴板应用。

## 实施策略
按 P0 → P1 → P2 优先级分阶段实现，每个阶段可独立测试验证。

---

## 阶段 1: 项目初始化与基础配置

### 任务 1.1: 创建项目结构
**目标**: 初始化项目目录和配置文件

**文件操作**:
- 创建 `package.json`
- 创建 `.env.example`
- 创建 `.dockerignore`
- 创建目录: `server/`, `public/`, `data/`, `logs/`

**验收标准**:
- 目录结构符合设计文档第 6 节
- package.json 包含所有依赖（设计文档第 11 节）

---

### 任务 1.2: Docker 配置
**目标**: 创建 Docker 部署文件

**文件操作**:
- 创建 `Dockerfile`
- 创建 `docker-compose.yml`

**验收标准**:
- Dockerfile 使用 node:18-alpine
- 端口映射为 3101
- 数据和日志目录正确挂载

---

## 阶段 2: P0 - MVP 核心功能

### 任务 2.1: 数据存储模块
**目标**: 实现原子写入的文件存储

**文件操作**:
- 创建 `server/storage.js`

**实现要点**:
- 读取/写入 JSON 文件（UTF-8）
- 原子写入：临时文件 + fs.renameSync
- 数据校验：版本号、条目数量（≤1000）、内容长度（≤100KB）
- 初始化空数据文件

**验收标准**:
- 导出 `loadData()` 和 `saveData(data)` 函数
- 并发写入不会导致数据损坏
- 数据格式符合设计文档第 3 节

---

### 任务 2.2: JWT 认证模块
**目标**: 实现密码验证和 JWT token 生成

**文件操作**:
- 创建 `server/auth.js`

**实现要点**:
- 验证密码（环境变量 CLIPBOARD_PASSWORD）
- 生成 JWT token（7 天有效期）
- 验证 JWT token 中间件

**验收标准**:
- 导出 `verifyPassword(password)` 返回 token
- 导出 `verifyToken(token)` 返回验证结果
- Token 包含过期时间

---

### 任务 2.3: Express 服务器基础
**目标**: 创建 HTTP 服务器和静态文件服务

**文件操作**:
- 创建 `server/index.js`

**实现要点**:
- Express 应用初始化
- 静态文件服务（public 目录）
- POST `/auth` 端点（密码验证）
- 环境变量读取（PORT=3101）

**验收标准**:
- 服务器监听 3101 端口
- `/auth` 返回 JWT token 或 401
- 可访问静态文件

---

### 任务 2.4: Socket.io 集成与认证
**目标**: 实现 WebSocket 连接和身份验证

**文件操作**:
- 修改 `server/index.js`

**实现要点**:
- Socket.io 服务器初始化
- 连接时验证 JWT token
- 认证成功推送全量数据
- 认证失败断开连接

**验收标准**:
- 客户端必须提供有效 token 才能连接
- 连接成功后收到 `sync_data` 事件
- 无效 token 返回 `auth_failed`

---

### 任务 2.5: WebSocket 事件处理
**目标**: 实现增删改操作和广播

**文件操作**:
- 修改 `server/index.js`

**实现要点**:
- `add_item`: 添加条目 + 广播
- `delete_item`: 删除条目 + 广播
- `update_item`: 更新条目 + 广播
- 每次操作后调用 `saveData()`

**验收标准**:
- 操作成功后所有客户端收到广播
- 数据持久化到文件
- 生成 UUID 作为条目 ID

---

### 任务 2.6: 前端页面结构
**目标**: 创建 HTML 页面和基础样式

**文件操作**:
- 创建 `public/index.html`
- 创建 `public/style.css`

**实现要点**:
- 登录表单（密码输入）
- 剪贴板条目列表容器
- 添加条目输入框
- 引入 Socket.io-client CDN

**验收标准**:
- UTF-8 编码声明
- 响应式布局
- 简洁美观的 UI

---

### 任务 2.7: 前端 WebSocket 逻辑
**目标**: 实现客户端 Socket.io 通信

**文件操作**:
- 创建 `public/app.js`

**实现要点**:
- 登录逻辑（POST `/auth` 获取 token）
- Socket.io 连接（携带 token）
- 监听 `sync_data` 更新 UI
- 监听 `item_added/deleted/updated` 增量更新
- 发送 `add_item/delete_item/update_item` 事件

**验收标准**:
- 登录成功后建立 WebSocket 连接
- 实时同步所有客户端操作
- 断线自动重连

---

### 任务 2.8: 文本折叠功能
**目标**: 实现前端折叠/展开逻辑

**文件操作**:
- 修改 `public/app.js`
- 修改 `public/style.css`

**实现要点**:
- 超过 3 行自动折叠
- 点击切换展开/收起
- localStorage 保存折叠状态

**验收标准**:
- 长文本默认折叠
- 折叠状态在刷新后保持
- 平滑的展开/收起动画

---

## 阶段 3: P1 - 生产就绪

### 任务 3.1: 速率限制
**目标**: 防止滥用和 DDoS

**文件操作**:
- 创建 `server/rateLimit.js`
- 修改 `server/index.js`

**实现要点**:
- express-rate-limit 中间件
- 每秒最多 10 次操作
- 超限返回 429 错误

**验收标准**:
- 快速操作会被限制
- 返回友好的错误信息

---

### 任务 3.2: 安全头和 Helmet
**目标**: 添加安全 HTTP 头

**文件操作**:
- 修改 `server/index.js`

**实现要点**:
- 使用 helmet.js 中间件
- 配置 CSP、X-Frame-Options

**验收标准**:
- 响应包含安全头
- 通过安全扫描工具检查

---

### 任务 3.3: 日志系统
**目标**: 记录操作和错误

**文件操作**:
- 修改 `server/index.js`

**实现要点**:
- winston 日志配置
- 记录所有操作（add/delete/update）
- 记录错误和警告
- 日志文件轮转

**验收标准**:
- logs/app.log 包含操作记录
- 错误信息完整可追溯

---

### 任务 3.4: 错误处理完善
**目标**: 统一错误处理

**文件操作**:
- 修改 `server/index.js`
- 修改 `public/app.js`

**实现要点**:
- 服务端 try-catch 包裹所有操作
- 客户端显示友好错误提示
- 网络异常自动重试

**验收标准**:
- 任何错误都有日志记录
- 用户看到清晰的错误信息
- 不会因错误导致服务崩溃

---

## 阶段 4: P2 - 优化增强

### 任务 4.1: 自动备份
**目标**: 定期备份数据文件

**文件操作**:
- 修改 `server/storage.js`

**实现要点**:
- 每次写入时创建 .bak 备份
- 文件损坏时自动恢复

**验收标准**:
- data/clipboard.json.bak 存在
- 损坏文件可恢复

---

### 任务 4.2: 前端分页显示
**目标**: 优化大量条目性能

**文件操作**:
- 修改 `public/app.js`
- 修改 `public/style.css`

**实现要点**:
- 超过 100 条时分页
- 虚拟滚动或懒加载

**验收标准**:
- 1000 条数据流畅显示
- 滚动无卡顿

## 测试验证

### 功能测试
1. 打开两个浏览器窗口，验证实时同步
2. 输入中文、emoji、特殊字符
3. 断网后重连，验证数据一致性
4. 错误密码验证被拒绝

### 压力测试
1. 快速添加 100 条数据
2. 10 个并发连接同时操作
3. 添加接近 100KB 的大文本

### 部署测试
1. `docker-compose up -d` 启动成功
2. 容器重启后数据不丢失
3. 日志正常输出
